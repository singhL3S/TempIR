About the Projects

We provide opportunity for students to work on small projects to put into action the concepts learnt in the lecture. Projects are not mandatory but we highly recommend participating in projects to get the most out of the lecture. Each project will be accompanied with a concrete task definition, data and initial references (see below). Students are required to work on them with a maximim group size of three, and co-ordinate and report intermediate results to the associated mentors. All projects will be finally integrated into our temporal retrieval system codebase for the lecture. We will use Terrier, which is an open source search engine, for most of the projects.

A successfully completed project should (a) solve the task definition, (b) be able to be integrated into the final temporal search codebase (c) contain an accompanying report of 1 page documenting the results.

To acknowledge successful participation in projects we give out badges according to the performance in successfully completed projects. Each badge earns a 0.3 grade improvement on the final grade of the final exam. Note that the grade improvement will only be awarded to those students who pass the final exam. A successfully completed project get at least two badges, and the maximum number of badges earned can be four. Please enter your preferences in the doodle here.

Temporal Indexing

Indexing Time and Phrases

Traditional search engine implementations index single words along with select textual features. The support for temporal features are central to build a temporal search system. We would want to treat time as a first class citizen and add support for time in our indexing system. This would enable us to efficiently retrieve documents which are relevant for a given period of time. For this project, we require that students understand the indexing process, add support for time as a field, and add querying functionality which filters documents given a time period.

Also, indexing common phrases adds efficient access to the temporal collection for phrase queries. In this project we would also require the students to support indexing of multi-word phrases to improve phrase-query processing.

  • References

"Phrase Query Optimization on Inverted Indexes." Avishek Anand, Ida Mele, Srikanta Bedathur, Klaus Berberich. in CIKM 2014.

"Temporal Index Sharding for Space-Time Efficiency in Archive Search." Avishek Anand, Srikanta Bedathur, Klaus Berberich, Ralf Schenkel. in SIGIR 2011.

Temporal Retrieval Models

Aspect Mining for Diversity Aware Search

Diversity is a important requirement for multi-faceted ambiguous queries like "jaguar", "olympic games", "prism" etc. Moreso, when we are dealing with temporal collections, queries mean different things at different points in time. Retrieval models which take diversity of aspects as well as time of documents into account are hence central to a temporal retrieval system. In this project students are supposed to implement the retrieval models discussed in the lecture which take "diversity of aspects" into account explicity. However, a key component in modelling diversity is the identification of these aspects in the documents. These aspects are both topical and temporal. A document talking about the arab spring can be associated with topical aspects "revolution" or "the arab spring". On the other hand temporal aspects could be the publication times and temporal expressions associated with the documents.

  • References

“Multi-Dimensional Search Result Diversification.” Dou, Zhicheng, Sha Hu, Kun Chen, Ruihua Song, and Ji-Rong Wen. In WSDM ’11.

Temporal Analysis and Mining for Query Enrichments

Time-Series Analysis for Auto-Completions

To improve the search user experience it is imperative to provide the user with useful cues in the form of potential auto-completions. In this project, we are interested in using temporal patterns from different temporal sources like anchor texts to provide query auto-completions building on the concepts discussed in the lecture. More specifically we use time series analysis methods like forecasting, correlation and burst analysis to predict the most relevant auto-completions given an already typed in query. We evaluate the project both by the effectiveness and efficiency of the approach.

  • References

“Behavioral Dynamics on the Web: Learning, Modeling, and Prediction.” Radinsky, Kira, Krysta M. Svore, Susan T. Dumais, Milad Shokouhi, Jaime Teevan, Alex Bocharov, and Eric Horvitz. in TOIS 2013. in SIGIR 2011.

“Using Temporal Bursts for Query Modeling.” Peetz, Maria-Hendrike, Edgar Meij, and Maarten Rijke. In Information Retrieval 2014.

Query Suggestions for temporal collections

Another useful aid to improve the search user experience is to have query suggestions. After an end-user has partially input a query, intelligent search engines can suggest possible completions of the partial query to help end-users quickly express their information needs. All major web-search engines and most proposed methods that suggest queries rely on search engine query logs to determine possible query suggestions. Unfornately, query logs for long time periods are unavailable. We will explore mechanisms for generating query suggestions from the corpus without using query logs. We utilize the document corpus to ex- tract a set of candidate phrases. As soon as a user starts typing a query, phrases that are highly correlated with the partial user query are selected as completions of the partial query and are offered as query suggestions.

In this project, we are interested in exploiting these term frequencies over time (mined from the underlying collection) to provide year-wise query suggestions.

  • References

“Query Suggestions in the Absence of Query Logs.” Bhatia, Sumit, Debapriyo Majumdar, and Prasenjit Mitra. In SIGIR 2011.

“Mining Named Entities with Temporally Correlated Bursts from Multilingual Web News Streams.” Kotov, Alexander, ChengXiang Zhai, and Richard Sproat. in WSDM 2011.

Crawling

Change Frequency Estimation for Crawling

Web Crawling requires information about the change frequency of Web pages to allow efficient scheduling of future re-crawls. Usually this information is computed from data from past crawls, but there are also approaches to estimate the change frequency from the page itself. Information that can be used is e.g. the type of web page (homepage vs. blog entry vs. ...), the URL structure or the number and type of links. The goal of this project is to implement a change frequency estimator as a plugin to an existing Web crawler (e.g. Apache Nutch or Heritrix). The estimator should also be evaluated on an existing Web archive. Contact: gossen@l3s.de

  • References

“Looking at both the present and the past to efficiently update replicas of web content”, L. Barbosa, A. C. Salgado, F. de Carvalho, J. Robin, and J. Freire. In Workshop on Web Information and Data Management, 2005

“Effective change detection using sampling”, J. Cho and A. Ntoulas. In VLDB'02.
